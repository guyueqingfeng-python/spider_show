这个仓库记录了我学习Python爬虫一年的完整实践过程。不是那种整理好的、干干净净的示例代码，而是保留了每一步的真实状态。从最开始什么都不懂、照着别人的代码改，到慢慢能自己写一个完整的项目，再到后来意识到代码需要组织、需要复用、需要解耦。这个过程里写下的每一版代码都放在这里了。有写得好的地方，也有现在看来很幼稚的写法，但都是真实的学习痕迹。

为什么要把这些代码公开出来

其实最开始没想过要开源。只是自己写着练手，攒着攒着发现已经有好几个版本了。从最初的面向过程版本，到面向对象重构，再到尝试写自己的小框架，这条路径还挺清晰的。我觉得对于其他正在学爬虫的人来说，看一个成品项目有时候反而不知道怎么下嘴，但看一个人是怎么一步步走到成品的过程，可能更有参考价值。所以就把这些都整理出来，按顺序放好，让别人能看到一个普通学习者一年的真实进度。

项目里有什么

51job项目1.py 是第一个真正能跑通的项目。当时刚学完基础语法和selenium，想着练练手就去爬51job。这个版本完全是面向过程写的，从上到下几百行代码，所有逻辑都堆在一起，函数之间互相调用得挺乱。但当时能跑出结果来就已经很开心了。现在看来代码写得很糙，异常处理基本没有，变量名也是随便起的，不过这个版本对我来说意义挺大，毕竟是第一个完整的爬虫项目。

51job项目2.py 是过了一段时间之后的重构。当时已经开始觉得之前的代码没法维护了，改一个地方要动好几个函数，于是重新整理了一遍。这次改成了面向对象，把搜索、翻页、解析、保存这些功能都收进类的方法里，逻辑清晰了不少。命名也稍微规范了一点，开始注意代码的可读性。这个版本算是我从能写到会写的转折点。

spider_display框架.py 是第一次尝试写框架。当时用51job项目2写了一段时间，发现每次换个网站就要改好多代码，于是想着能不能把通用的部分抽出来，用配置文件来控制爬虫行为。这个版本把选择器、搜索关键词什么的都放进了json文件，核心代码基本不用动。虽然实现得挺简陋的，但好歹迈出了框架化的第一步。

spider_display框架2.py 是在上一版基础上的改进。之前那个版本虽然能用，但所有功能都挤在一个类里，改起来还是不太方便。这个版本把页面导航、数据抓取、文件保存拆成了不同的类，各管一摊，职责更清楚。整体结构上更接近Scrapy那种设计思路了，虽然只是模仿了个皮毛，但至少知道一个框架大概应该怎么组织了。

spider_dispaly_json.json 和 spider_dispaly_json2.json 是对应两个框架版本的配置文件模板。里面定义了url、选择器、字段映射这些东西，用户根据自己的需要填就行。目前只支持了基础功能，复杂的逻辑还是得改代码，不过对于简单的爬取任务已经够用了。

tools.py 是早期写的一些工具函数。当时刚学会把重复代码抽出来，就随手往这个文件里塞，什么都有，也没怎么整理。函数命名比较随意，注释也不全，属于能用但不太好看的状态。后来把这些工具单独拎出来成了一个仓库，这个tools.py就留在原地当个纪念了。

requirements.txt 是项目依赖列表。用pip install -r requirements.txt就能一次性装好所有需要的库，省得一个一个找。

怎么运行

第一步是装依赖。打开终端或者命令行，cd到项目文件夹，然后执行 pip install -r requirements.txt。网速慢的话可以加上清华源什么的。

第二步是运行。最简单的是直接跑 python 51job项目2.py，这个版本比较稳定，代码也相对规整，适合先跑起来看看效果。跑之前确保电脑上有Chrome浏览器，因为selenium需要调用它。如果Chrome版本太老可能需要更新一下。

如果想试框架版本，先打开 spider_dispaly_json2.json 看看，把url改成你想爬的网站，把选择器改成对应网站的定位方式，然后运行 python spider_display框架2.py。目前框架只测试过51job，换其他网站可能需要微调。

需要留意的是，爬太快可能会被网站封IP，代码里已经加了一些随机等待，但最好还是别在短时间内跑太多次，尤其不要在工作日的白天反复刷同一个网站。

依赖环境

Python 3.8 或更高版本。Chrome浏览器。其他依赖都在requirements.txt里，主要用到的库是selenium、requests、lxml、pandas这几个。

关于代码的一点说明

这个仓库是我个人学习的代码快照，不是生产级工具，也没有经过严格测试。有些代码写得比较随意，比如早期版本里到处都是拼音命名、到处是宽泛的异常捕获、注释也不完整。但每个版本都代表了当时我所能达到的最好水平，所以就原样保留了。如果您想看看正常一些的代码，推荐看51job项目2.py和spider_display框架2.py，这两个版本相对工整。

如果您发现了bug或者有改进建议，欢迎提issue，看到了会认真处理的。不过有些早期代码的写法我自己也知道问题在哪，但暂时没打算改，因为留着能看出来当时是怎么想的，改了就看不出来了。
